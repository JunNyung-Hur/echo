# echo v0.0.1

The first release of **echo** — a personal Second Brain that lives on your laptop.

Capture anything you think or hear — meetings, lectures, interviews, quick memos, talking to yourself — and an AI agent beside your note transcribes it and organizes it into clean notes you grow and refine by chatting. It's a single-user app — no account, no server.

## What you can do

- **Two kinds of notes, shaped to what you capture**
  - **Minutes** — record a meeting or lecture and echo transcribes it, then generates a structured write-up (decisions, action items, sections) sized to the content.
  - **Freeform** — a chat-first notepad. Jot text, or attach voice recordings and audio files right in the chat (record, pick a file, or drag-and-drop, several at once); echo transcribes each and merges it into your note, keeping what's already there and splitting different topics into sections.
- **An AI agent beside the note** — restructure, summarize, change the wording or style, fix a mis-heard word, or recover from a failed transcription, all by asking in plain language.
- **Bring your own models** — connect any OpenAI-compatible LLM and ASR endpoint, cloud (OpenAI, etc.) or local (vLLM).
- **And more** — version history with rollback, `#tag` + text search, and a full Korean / English UI.

## Install (Windows x64)

1. Download **`echo_0.0.1_x64-setup.exe`** below and run it.
2. **ffmpeg is bundled** — nothing else to install for audio. WebView2 is installed automatically if it's missing (Windows 11 already has it).
3. Open **Settings → AI models** and register an OpenAI-compatible **ASR** endpoint (transcription) and **LLM** endpoint (note generation/refinement). A cloud API key or a local server both work.

That last step is required — recording/transcription and note generation can't run until both endpoints are set.

See the [user guide](../user-guide.md) for a full walkthrough.

## Requirements

- Windows x64. (It runs on Windows 11 ARM via x64 emulation, but ARM is not a build target.)
- An OpenAI-compatible ASR endpoint and LLM endpoint that you provide.

## Notes & limitations

- Single-user by design — no accounts, no sync. Your data lives in a local SQLite database in your app-data folder.
- API keys are stored locally in plaintext in this version.
- Full-text search over note bodies isn't included yet (search covers titles, memos, locations, and tags).
- macOS / Linux builds aren't published in this release.

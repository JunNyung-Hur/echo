# echo v0.0.2

A quality + housekeeping update for **echo** — sharper note-taking, easier access to your files, and a tidier on-disk layout. Builds on v0.0.1.

## What's new

- **File archive, with one-click folder access** — the recording archive is now a **File archive (파일 보관함)**, and you can open a note's folder on disk straight from it. Available on both Freeform and Minutes notes (previously Freeform only).
- **Transcript at a glance on Minutes** — a **Transcript** button next to the version history opens the full transcript in a modal.
- **Chat that knows what's worth writing down** — Freeform chat now tells note content apart from conversation. Greetings, small talk, and questions are handled in chat and no longer get written into your note — only real content does. Ask "what did I just write?" and it answers from the note instead of recording the question.
- **Tidier storage** — each note's recordings, transcripts, and body now live under a single per-note folder, with shorter, cleaner paths; deleting a note removes its folder in one go.

## Install (Windows x64)

1. Download **`echo_0.0.2_x64-setup.exe`** below and run it.
2. **ffmpeg is bundled** — nothing else to install for audio. WebView2 is installed automatically if it's missing (Windows 11 already has it).
3. Open **Settings → AI models** and register an OpenAI-compatible **ASR** endpoint (transcription) and **LLM** endpoint (note generation/refinement). A cloud API key or a local server (vLLM) both work.

That last step is required — recording/transcription and note generation can't run until both endpoints are set.

See the [user guide](../user-guide.md) for a full walkthrough.

## Requirements

- Windows x64. (It runs on Windows 11 ARM via x64 emulation, but ARM is not a build target.)
- An OpenAI-compatible ASR endpoint and LLM endpoint that you provide.

## Notes & limitations

- Single-user by design — no accounts, no sync. Your data lives in a local SQLite database in your app-data folder.
- API keys are stored locally in plaintext in this version.
- Full-text search over note bodies isn't included yet (search covers titles, memos, locations, and tags).
- macOS / Linux builds aren't published in this release.
- Upgrading from v0.0.1: existing notes keep working; new notes use the tidied per-note folder layout.
